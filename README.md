# digital-environmental-impact-analysis

Web eco-design audit and carbon footprint analysis of Alibaba.com using EcoIndex, GreenIT, and CodeCarbon. This project evaluates the environmental impact of web applications through eco-design metrics and compares the efficiency of different computational implementations.

## Project Overview

This project aims to analyze and quantify the environmental footprint of web applications and computational algorithms by:

1. **Web Eco-Design Audit**: Using EcoIndex methodology to assess the environmental impact of web pages
2. **Algorithmic Comparison**: Benchmarking different implementations of matrix operations and attention mechanisms to compare their computational efficiency and carbon emissions
3. **Emissions Tracking**: Using CodeCarbon to measure real-time CO₂ emissions during computational tasks

### Key Objectives

- Evaluate PyTorch vs custom NumPy implementations for attention mechanisms
- Compare matrix multiplication approaches (PyTorch vs multithreaded custom implementation)
- Quantify the carbon footprint of computational operations
- Promote awareness of computational sustainability in deep learning

## Technologies & Tools

- **PyTorch**: Deep learning framework for tensor computations
- **NumPy**: Numerical computing library
- **CodeCarbon**: Real-time emissions tracking for computational workloads
- **EcoIndex**: Methodology for assessing web page environmental impact
- **Matplotlib**: Data visualization

## Requirements

Python 3.13+

## Installation

Install the Python dependencies from the project root:

```bash
pip install -r Requirements.txt
```

## Usage

Run the scripts from the project root:

```bash
# EcoIndex audit
python scripts/ecoindex/test_ecoindex.py https://www.alibaba.com

# Matrix multiplication benchmark
python scripts/matmul/benchmark_matmul.py

# Attention benchmark
python scripts/attention/benchmark_attention.py
```

## Scripts

### Ecoindex Audit
- `scripts/ecoindex/test_ecoindex.py`: Computes EcoIndex indicators for a target web page and estimates environmental impact using the GreenIT methodology.

### Attention Mechanism Benchmarks
- `scripts/attention/attention_base.py`: PyTorch implementation of scaled dot-product attention.
- `scripts/attention/attention_custom.py`: Custom NumPy-based scaled dot-product attention implementation optimized for comparison.
- `scripts/attention/benchmark_attention.py`: Comprehensive comparison of PyTorch vs custom attention implementations with emissions tracking.

### Matrix Multiplication Benchmarks
- `scripts/matmul/matmul_concurrent.py`: Custom multithreaded matrix multiplication implementation.
- `scripts/matmul/benchmark_matmul.py`: Comparison of PyTorch matrix multiplication with custom concurrent implementation, including statistical analysis (mean, std dev) and 2D visualization.

## Results

The `results/` folder contains generated outputs from the benchmark and EcoIndex scripts.

CSV files:

- `results/data/comparison_results.csv`: generated by `scripts/attention/benchmark_attention.py`. It stores execution time and estimated emissions for the attention benchmark.
- `results/data/comparison_pytorch_concurrent.csv`: generated by `scripts/matmul/benchmark_matmul.py`. It stores execution time and estimated emissions for the matrix multiplication benchmark.
- `results/data/emissions.csv`: generated by CodeCarbon during script execution. It stores emissions measurements collected while the experiments run.

Figures:

- `results/figures/attention-emissions-comparison.png`: generated by `scripts/attention/benchmark_attention.py`. 2D comparison chart showing execution time and emissions impact of PyTorch vs custom attention implementations.
- `results/figures/matrix-multiplication-emissions-comparison.png`: generated by `scripts/matmul/benchmark_matmul.py`. Comparative analysis of PyTorch vs multithreaded matrix multiplication performance and emissions.

Run the scripts from the project root so generated files are written to the expected `results/data/` and `results/figures/` folders.

## Key Findings

- PyTorch implementations generally show lower execution times for small to medium matrix sizes
- Custom implementations may have higher emissions due to CPU-intensive operations vs GPU optimization
- Emissions tracking reveals significant variations based on matrix size and implementation choice
- Regular benchmarking helps identify optimization opportunities for sustainable computing

## Environmental Impact Metrics

All experiments track:
- **Execution time** (seconds)
- **CO₂ emissions** (kg CO₂e and Maybe μg CO₂e )
- **Power consumption** (estimated)
- **Energy consumption** (kWh)
